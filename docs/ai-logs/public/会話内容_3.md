対応入れました。Controller から infra の `BookEntity` 依存を外し、Service からはドメインモデル `Book` を返す構造に変更しています。あわせて `BookController#toResponse` の `null -> 0L` は廃止し、「未採番は異常」として例外にしました。

- **責務分離（Controller が Entity を扱わない）**
  - ドメインモデル追加: Book.java
  - Service の戻り値を `BookEntity` → `Book` に変更（Entity→Model変換は Service 内に閉じる）: BookService.java

  - `BookService#toModel()` で `entity.getBookId() == null` の場合は `IllegalStateException("bookId is not assigned")` を送出（= 未採番は異常扱い）
  - これにより `BookController#toResponse()` 側の「0埋め」は不要になりました（`Book.bookId()` は `long`）

- **テスト追従 & 実行**
  - Service/Controller テストを `Book` 返却に追従: BookServiceTest.java, BookControllerWebMvcTest.java
  - `mvn -q test` は `EXIT:0` で通過済み
